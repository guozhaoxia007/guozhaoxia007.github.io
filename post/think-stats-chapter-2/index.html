2 分布式

2.1 Historgrams

描述变量的最好方法之一是报告数据集中出现的值，以及每个值出现多少次。这种描述称为变量的分布。

分布最常见的表示是直方图，它是显示每个值的频率的图。在这种情况下，“频率”是指值出现的次数。

在Python中，计算频率的一个有效方法是用字典。定义一个序列，t

hist = {}
for x in t:
    hist[x] = hist.get(x, 0) + 1

字典的结果是值和频率的映射。或者，你可以使用集合模块中定义的计数器类：

from collections import Counter
counter = Counter(t)

Counter计数器对象的结果，是字典的一个子类。

另一个选项是使用pandas模块里的value_counts。在这本书里我创建了一个类，Hist,表示直方图，并提供操作他们的方法。

2.2 Representing histograms代表直方图

Hist构造函数可以采取一个序列，字典，pandas系列，或其它Hist. 你可以实例化一个Hist对象：

>>> import thinkstats2
>>> hist = thinkstats2.Hist([1, 2, 2, 3, 5])
>>> hist
Hist({1: 1, 2: 2, 3: 1, 5: 1})

Hist对象提供Freq, 传入一个值返回它的频率：

hist.Freq(2)
2

括号运算符跟上面不一样：

>>> hist[2]
2

如果你想找一个从没出现的值，序列是0.

>>> hist.Freq(4)
0

Values返回一个未排序的列表：

>>> hist.Values()
[1, 5, 3, 2]

要依次遍历这些值，可以使用内置函数排序sorted:

for val in sorted(hist.Values()):
    print(val, hist.Freq(val))

或者你可以使用Items遍历值-频率：

for val, freq in hist.Items():
    print(val, freq)

2.3 Plotting histograms

在这本书里我写了一个模块thinkplot.py，它提供绘制Hists和其它对象的函数。它是基于pyplot, 是matplotlib包的一部分。见0.2节关于matplotlib的安装信息。

用thinkplot绘制hist:

>>> import thinkplot
>>> thinkplot.Hist(hist)
>>> thinkplot.Show(xlabel='value', ylabel='frequency')

你可以阅读thinkplot文档，在 http://greenteapress.com/thinkstats2/thinkplot.html

2.4 NSFG variables

现在让我们说回NSFG数据。这章的代码在first.py中。0.2节说明了下载和运行代码的信息。

当你开始处理一个新的数据集，我建议你去探索你计划一次使用一个的变量，看直方图是一个好方法。

在1.6节我们将agepreg从世纪转成年，并合并birthwgt_lb和birthwgt_oz到一个单独的数字totalwgt_lb上。在这节我将使用这些变量演示直方图的一些特性。

从读取数据和选择活产记录开始：

preg = nsfg.ReadFemPreg()
live = preg[preg.outcome == 1]

括号里的表达式是一个布尔系列,它从DataFrame选了一些行又返回一个新的DataFrame. 接下来我生成和绘制了活产的birthwgt_lb的直方图。

hist = thinkstats2.Hist(live.birthwgt_lb, label='birthwgt_lb')
thinkplot.Hist(hist)
thinkplot.Show(xlabel='pounds', ylabel='frequency')

当传给Hist的参数是一个pandas Series，nan值会被忽略。label是一个字符串，当Hist绘制出来时会出现在图例上。

图2.1展示出了结果。最常见的值，叫做mode,是7磅。分布大致呈钟形，是分布图的一般形状，也叫作高斯分布图。但与正态分布不同，这种分布是不对称的，它有一个尾巴，向左延伸比向右延伸更远。

图2.2展示birthwgt_oz的直方图，是婴儿体重的ounces部分。理论上我们希望这个分布是均匀的；即所有的值都有相同的频率。事实上，0要比其它值更常见，1和15更少见，可能因为答卷人对接近整数的婴儿体重取整了。

图2.3展示了agepreg的直方图，母亲妊娠结束时的年龄。模式是21年。分布大致是钟形的，但在这种情况下，尾巴延伸到右边比左边更远。大部分母亲是20多岁，少部分在30多岁。

图2.4展示了prglngth的直方图，即孕周的长度。目前为止最常见的是39周。左边的尾巴比右边的长；早产婴儿很常见，但是怀孕很少超过43周，这种情况下医生经常会干预。

2.5 Outliers极端值

看直方图，很容易看出最常见的值和分布图的形状，但是不常见的值并不总是可见的。

在进行之前，检查异常是一个好方法，异常值可能是测量和记录中的错误，也可能是罕见事件的精确报告。

Hist提供了Largest和Smallest方法，值入整数n,返回n个最大或最小的值：

for weeks, freq in hist.Smallest(10):
    print(weeks, freq)

在活产的怀孕长度的列表中，10个最小的值是[0, 4, 9, 13, 17, 18, 19, 20, 21, 22]. 小于10周的值确定是错误的，比较合理的解释是极端值没有正确的编码。高于30周的值可能是合理的。10和30周之间很难确认；一些值可能是错误的，但是一些代表了早产。

在列表的另一端，最高值是：

weeks count
43 148
44 46
45 10
46 1
47 1
48 7
50 2

如果怀孕超过42周，大部分医生建议引产，所以一些lqta的值是令人惊讶的。特别是，50周看起来在医学上是不可能的。

处理极端值最好的方式取决于”领域知识”，即数据来自何处以及他们的含义。这取决于你打算执行什么分析。

在这个例子中，初衷问题是第一个婴儿是否早（或晚）。当人们问这个问题时，他们通常对足月妊娠感兴趣，因此我将着重于怀孕超过27周的分析。

2.6 First babies第一胎

现在我们可以比较第一胎和其它胎的怀孕长度分布图。我用birthord把DataFrame中活产区分出来并计算他们的直方图：

firsts = live[live.birthord == 1]
others = live[live.birthord != 1]

first_hist = thinkstats2.Hist(firsts.prglngth)
other_hist = thinkstats2.Hist(others.prglngth)

然后我绘制了同样X轴的直方图：

width = 0.45
thinkplot.PrePlot(2)
thinkplot.Hist(first_hist, align='right', width=width)
thinkplot.Hist(other_hist, align='left', width=width)
thinkplot.Show(xlabel='weeks', ylabel='frequency',
               xlim=[27, 46])

thinkplot.PrePlot需要传入我们计划绘制直方图的数量；它用这个信息决定一个颜色集合。

thinkplot.Hist通常用align = ‘center’所以每个柱处在它的值的中间。在这张图里，我用了align = ‘right’和 align = ‘left’在值的两边放置相应的柱子。

width=0.45，两条柱子的总宽是0.9，在每一对柱子间留一些空白。

最后，我调整X轴只显示27到46周之间的数据。图2.5展示了结果。

直方图是有用的因为他们直接显示出了最常见的值。但他们不是比较两个分布图的最好的选择。在这个例子中，”first babies”比其它胎要少，所以直方图中一些明显差异是由于样本大小造成的。在下一章中我们利用概率质量函数来解决这个问题。

2.7 Summarizing distributions总结分布

一个直方图是一个样例分布的完整描述；即给出一个直方图，我们可以重新组织样例的值（不按他们的顺序）。

如果分布的细节是重要的，那么可能需要呈现直方图。但是我们经常需要用一些描述性统计来总结分布。

我们可能要报告的一些特点是： - 中心趋势：值会倾向于围绕某一个特定点聚集吗？ - 模式：有多个集群吗？ - 范围：值有多大的可变性？ - 尾巴：当我们离开某一模式时概率会下降的有多快？ - 离群值：是否有远离模式的极端值？

回答这些问题的统计设计叫做summary statistics总结统计。目前最常见的汇总统计是mean平均值，它用来描述分布的中心趋势。

如果你有n个值的样例，x_i,平均值，\overline{x}, 是值的总和除以值的个数。换句话：

“mean”和”average”通常可以直接互换，但是我做了一些区分： - “mean”，样本的”平均值”是用前面公式计算的摘要统计量。 - “average” “平均值”是你可以用来描述一个中心趋势的几个汇总统计数据之一。

有时平均值是一个数据值集合的好的描述。比如，苹果几乎都是同样的大小（至少超市是卖的一样的）。所以如果我买6个苹果，总重3磅，有理由说他们每个重0.5磅。

但南瓜品种更多。假如我在花园里种了几个品种，有一天我收获了三种南方，每个重1磅，两个南瓜派分别重3磅，一个大西洋GiantR南瓜重591磅。这个样本的平均值是100磅，但如果我告诉你：我菜园里的南瓜平均重100磅，那将是误导性的。在这个例子中，没有有意义的平均值因为没有典型的南瓜。

2.8 Variance方差

如果没有一个数据能总结南瓜的我不是，我们可以用两个数字表达会好一点：均值和方差。

方差是一种摘要统计量，用于描述分布的变化或分布。一组值的方差是：

项x_i - \overline{x}被称为”偏离平均值”，所以方差是均方差。方差的平均根，S，是标准偏差。

如果你有经验，你可能会看到一个分母是n-1的方差的公式，而不是n. 这个统计是用来估计人口的方差。我们将在第8章讲到这一点。

Pandas数据结构提供计算均值、方差和标准偏差的方法：

mean = live.prglngth.mean()
var = live.prglngth.var()
std = live.prglngth.std()

在所有活产中，怀孕长度的均值是38.6周，标准偏离是2.7周，即我们期望2-3周的偏离是常见的。

怀孕长度的方差是7.3，这很难解释，尤其单位是周的平方，或平方周。在一些计算中方差是有用的，但不是一个好的汇总统计量。

2.9 Effect size规模效应

规模效应是一个摘要统计量，用于描述规模大小。比如，为了描述两组之间的区别，一个显著的选择是均值差。

第一胎的妊娠周期均值是38.601；其他胎是38.523.差值是0.078周，13小时多。作为典型怀孕周期的一小部分，这种差异约为0.2%。

如果我们假定这个估计是准确的，那么这种差别就没有实际的后果。事实上，如果不观察大师的怀孕，就不可能有人注意到这种差别。

另一种传达规模大小的方法是比较组之间的差异和组内的变异性。Cohen’s d是就是用来做这种统计的，它的定义：

Python代码：

def CohenEffectSize(group1, group2):
	diff = group1.mean() - group2.mean()
	var1 = group1.var()
	var2 = group2.var()
	n1, n2 = len(group1), len(group2)
	pooled_var = (n1 * var1 + n2 * var2) / (n1 + n2)
	d = diff / math.sqrt(pooled_var)
	return d

在这个例子中，均值差是0.029标准偏差，是很小的。从这个角度来看，男女身高差是1.7个标准差。

2.10 Reporting results 报告结果

我们有好几种方式描述第一胎和其他胎怀孕周期的差别。那我们怎么报告这些结果呢？

答案取决于谁在问这个问题。科学家可能对任何真实的效果感兴趣，无论多小。一个医生可能只关心具有临床意义的效果，也就是说，影响治疗决策的差异。孕妇可能对与她有关的结果感兴趣，比如早或晚的概率。

你报告结果的方式也取决于你的目标。如果你试图证明效果的重要性，你可以选择强调差异的汇总统计数据。如果你想让病人放心，你可以选择统计数据，把这些差异放在上下文中。

当然你的决定也应该以职业道德为指导。有说服力是可以的；你应该设计一个可以清晰讲述故事的统计报告和可视化的。但你也应该尽最大努力使你的报告诚实，承认不确定性和局限性。

2.12 Glossary词汇表

  • 分布：在样本中某个值的出现和频率。
  • 直方图：值和频率的映射，或展示该映射结果的图。
  • 频率：样本中某个值出现的次数。
  • 模式：样本是常见的值
  • 正态分布：理想情况是一个钟形分布；也称为高斯分布。
  • 均匀分布：所有值具有相同频率的分布。
  • 尾巴：分布在高低两极的部分。
  • 集中趋势：样本或人口的特征；直观地说，它是一个平均值或典型值。
  • 离群值：远离集中趋势的值。
  • 传播：分布中的值如何展开的度量。
  • 汇总统计：量化某一分布的某一方面，如集中趋势或散布的统计数据。
  • 方差：常用于量化传播的摘要统计量。
  • 标准差：方差的平方根，也用作度量散布。
  • 效应大小：一种摘要统计量，用于量化不同群体之间的效应的大小。
  • 临床意义：一个结果，如群体之间的差异，在实践中是相关的。
